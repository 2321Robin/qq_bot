# S2 精灵捕捉计数器设计

## 背景

现有 QQ 机器人已经支持洛克王国世界精灵图鉴查询和技能查询。新功能需要提供一个个人捕捉计数器，用于记录每个用户在当前赛季中每只精灵出了几只，并可标记其中几只是异色。

## 目标

- 新增 `/计数` 命令。
- 支持按用户、群、赛季隔离统计。
- 当前赛季默认为 `S2`。
- 支持普通计数和异色计数。
- 支持查看当前用户当前赛季汇总。
- 数据持久化到独立 SQLite 数据库。

## 非目标

- 不实现候选精灵列表。
- 不实现撤销。
- 不实现重置。
- 不要求计数精灵名必须存在于本地图鉴。
- 不生成图片卡片，先返回纯文本。

## 用户命令

新增命令：

```text
/计数
/计数 迪莫
/计数 异色 迪莫
```

行为：

- `/计数`：显示当前群、当前用户、当前赛季的汇总。
- `/计数 迪莫`：将 `迪莫` 的普通捕捉数加 1，并返回该精灵和总计当前统计。
- `/计数 异色 迪莫`：将 `迪莫` 的异色捕捉数加 1，同时总捕捉也加 1，并返回该精灵和总计当前统计。

示例返回：

```text
S2 捕捉计数器
总捕捉：12 | 异色：2
迪莫：5（异色 1）
喵喵：7（异色 1）
```

```text
S2 迪莫 +1
当前：3 | 异色：0
总捕捉：8 | 总异色：1
```

```text
S2 异色 迪莫 +1
当前：4 | 异色：1
总捕捉：9 | 总异色：2
```

## 数据存储

使用独立 SQLite 文件：

```text
data/roco_counter.sqlite3
```

新增配置：

```text
ROCO_COUNTER_PATH=data/roco_counter.sqlite3
ROCO_COUNTER_SEASON=S2
```

核心表字段：

```text
group_id INTEGER NOT NULL
user_id INTEGER NOT NULL
season TEXT NOT NULL
pet_name TEXT NOT NULL
normal_count INTEGER NOT NULL DEFAULT 0
shiny_count INTEGER NOT NULL DEFAULT 0
updated_at TEXT NOT NULL
```

唯一键：

```text
group_id, user_id, season, pet_name
```

总捕捉数按 `normal_count + shiny_count` 计算。异色记录不写入 `normal_count`，只增加 `shiny_count`。

## 代码结构

新增服务模块：

```text
src/qq_bot/services/roco_counter.py
```

职责：

- 初始化 SQLite 表。
- 记录普通捕捉 +1。
- 记录异色捕捉 +1。
- 读取某只精灵统计。
- 读取当前用户当前赛季汇总。
- 格式化记录成功和汇总文本。

新增插件模块：

```text
src/qq_bot/plugins/roco_counter.py
```

职责：

- 注册 `/计数` 命令。
- 复用 `settings.group_allowed(event.group_id)` 群权限。
- 解析命令参数。
- 调用 `RocoCounterStore` 或同等服务对象。
- 使用现有 `finish_with_send_errors_logged` 发送结果。

更新帮助文本：

```text
/计数 迪莫 或 /计数 异色 迪莫 - 记录 S2 精灵捕捉数量
```

## 参数解析

- 空参数表示查询汇总。
- 参数以 `异色` 开头时，后续文本作为精灵名。
- 其他非空参数整体作为普通精灵名。
- 多个连续空白会被压缩，避免 `/计数    迪莫` 或 `/计数 异色   迪莫` 解析失败。
- `/计数 异色` 因缺少精灵名，返回用法说明。

## 错误处理

- 未授权群直接返回，不发送消息。
- 缺少异色精灵名时返回用法说明。
- 数据库目录不存在时自动创建。
- 精灵名不在本地图鉴中也允许记录。

用法说明：

```text
用法：/计数 迪莫 或 /计数 异色 迪莫；发送 /计数 查看汇总。
```
